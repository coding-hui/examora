# Examora Internal Modules

内部技术规划统一按模块组织，避免使用 roadmap/backlog/roles 这类横向视角文件。

建议开发顺序：

1. `01-foundation.md`
2. `02-auth-and-access.md`
3. `03-subject-question-bank.md`
4. `04-paper-and-exam-publishing.md`
5. `05-candidate-exam-flow.md`
6. `06-programming-judge.md`
7. `07-audit-and-operations.md`

每个模块文档包含：

- 目标
- 范围
- 数据表
- API
- 页面/客户端
- 实现备注

后续如果继续细化，也直接追加到对应模块，不再新增横向规划文档。
